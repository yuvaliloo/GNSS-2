# GNSS-Denied Drone Localization Pipeline

This project implements a hierarchical visual localization pipeline to estimate a drone's trajectory in environments where GNSS data is unavailable or unreliable. It fuses visual odometry (via hloc and COLMAP) with raw onboard telemetry to generate a high-precision flight path.

## Prerequisites

- Python 3.10+
- Git
- System Dependencies: Ensure you have `ffmpeg` and `colmap` installed on your system path.

## Setup Instructions

### 1. Initialize the Repository

Clone the repository and initialize the submodules required for the localization engine:

```bash
git clone <repository-url>
cd <project-folder>
git submodule update --init --recursive
```

### 2. Environment Configuration

Set up a virtual environment to isolate the project dependencies.

Create and activate a virtual environment:

```bash
python -m venv venv
```

#### Windows

```powershell
venv\Scripts\activate
```

#### Linux/macOS

```bash
source venv/bin/activate
```

### 3. Install Dependencies

Install all required Python libraries:

```bash
pip install -r requirements.txt
```

### 4. Data Preparation (Crucial)

Before running the pipeline, you must provide the raw drone flights and telemetry. Place them in the exact folders below with these exact names to ensure the scripts run without crashing.

#### Training Data (Building the Map)

```text
data/training/v11.mp4
data/training/v11.srt
```

- `v11.mp4` — The raw video stream of the training flight.
- `v11.srt` — The corresponding DJI telemetry file (IMU/GPS data).

#### Testing Data (GNSS-Denied Live Flight)

```text
data/testing/v12.mp4
data/testing/v12.srt
```

- `v12.mp4` — The raw video stream of the test flight.
- `v12.srt` — The corresponding DJI telemetry file (IMU/GPS data).

## Running the Pipeline

### 1. Extract Training Frames

First, extract a sequence of images (1 FPS) from the baseline video to be used for the visual database:

```bash
python extract_train.py
```

### 2. Build the Map

If you are starting with a new environment, run the map-builder script to generate the 3D reference database using SuperPoint features and COLMAP:

```bash
python build_map.py
```

### 3. Run Visual Odometry Navigation

Execute the localization pipeline to fuse visual AI matching data with the drone's IMU telemetry:

```bash
python navigate.py
```

**Outputs:**

- `outputs/estimated_trajectory.srt`
- Comparison KML file

### 4. Optical Target Tracking (Finale)

This finale script uses the trajectory generated by the previous step, applies the drone's pitch and yaw telemetry, and computes the exact ground target coordinates the camera is looking at.

```bash
python target_tracker.py
```

**Outputs:**

- `outputs/FINAL_ASSIGNMENT_MAP.kml`

> Note: We only extracted 1 out of every 30 frames (1-second intervals since the video is 30 FPS) to optimize processing time.
